# 数学口算题生成器 — 开发文档

## 项目概述

一个纯前端网页应用，用于生成小学数学口算练习题PDF。用户选择题型、设置页数和每页题数，一键生成A4排版的PDF文件，可打印或保存。

- **线上地址**：https://xiaosong746-png.github.io/math-worksheet/
- **本地文件**：`math-worksheet/index.html`
- **技术栈**：HTML + CSS + JavaScript（单文件，零外部依赖）
- **托管平台**：GitHub Pages（免费）

```
┌──────────────────────────────────────────┐
│              index.html（单文件）          │
│  ┌─────────┐ ┌────────┐ ┌────────────┐  │
│  │  HTML   │ │  CSS   │ │ JavaScript │  │

│       ↓           ↓            ↓         │
│   用户操作    移动端适配   题库+PDF生成    │
│                      ↓                   │
│              window.print()              │
│           （浏览器原生打印→另存为PDF）      │
└──────────────────────────────────────────┘
```

- **无服务器**：所有计算在用户浏览器中完成
- **无数据库**：题库在内存中生成，使用记录存在localStorage
- **PDF生成**：将打印内容写入Blob新窗口（自带正确标题），调用浏览器原生打印→另存为PDF。不依赖任何第三方库
- **无外部依赖**：纯原生HTML/CSS/JS，不需要CDN

## 出题算法（分层抽样）

### 旧算法（v2.x 之前）

纯随机抽取：题库洗牌 → 取前N道 → 完成。**问题**：一页100题可能出现结果值严重偏斜（如大量题目集中在结果10-12，0-5和17-20几乎没有）。

### 新算法（v3.0，2026-07-11）

```
1. 按结果值分桶
   例：20以内加法 → 21个桶（结果0, 1, 2, ..., 20）

2. 每桶内随机洗牌

3. 轮询从各桶均匀抽取
   每轮：桶0取1道 → 桶1取1道 → ... → 桶20取1道
   5轮 = 105道，取前100道

4. 近重复过滤
   "3+5=" 和 "5+3=" 同结果、同操作数集合 → 只保留一个

5. 跨题型混合洗牌 → 分页
```

### 核心函数

| 函数 | 作用 |
|------|------|
| `generateAllQuestions(key)` | 枚举某题型所有合法算式 |
| `getResult(key)` | 从算式key提取计算结果（如 "3+5" → 8） |
| `getMaxExprChars(types)` | 估算选中题型最长算式字符数，用于字号计算 |
| `selectQuestions(types, count)` | 分层抽样主函数 |
| `calculateLayout(count, types)` | 计算列数/字号/行高/列宽 |

### 去重与循环机制

- **页内去重**：分层抽样 + 近重复过滤，确保同一页覆盖面广且不浪费题位
- **跨次去重**：已用题目存入 `localStorage`（key: `math_worksheet_used`），下次生成时跳过
- **循环重置**：某题型题库不够时，自动重置并重新洗牌，继续抽取
- **静默运行**：全程无提示，用户无感知

## PDF生成机制

### 方式

使用**浏览器原生打印**（`window.print()`），用户选择"另存为PDF"保存。

### 流程

```
1. 分层抽样选出题目
2. 生成A4排版HTML → 放入隐藏的print-area
3. 提取print-area内容 + 所有CSS样式
4. 构建完整HTML文档（<title>设为"日期+口算题"）
5. 创建Blob → 打开新窗口（标题从创建即正确）
6. 新窗口自动调用window.print()
7. 用户选"另存为PDF"保存
```

### 为什么用Blob新窗口？

Android Chrome / Safari 打印PDF时，文件名来自**文档创建时的 `<title>` 标签**，动态修改 `document.title` 不生效。将打印内容放入全新窗口，标题从一开始就是正确的，确保PDF文件名统一为 `2026-07-11口算题` 格式。

## 排版规则

### 始终4列，字号自适应

| 题型类别 | 每页题数 | 列数 | 字号 | 说明 |
|----------|----------|------|------|------|
| 20以内加减法 | 100 | 4列 | **21pt** | 算式最短，字号最大 |
| 20以内进位/退位 | 100 | 4列 | **19pt** | 两位数运算 |
| 20以内连加/连减 | 100 | 4列 | **17pt** | 三项运算，表达式最长 |
| 50以内加减法 | 100 | 4列 | **17pt** | 数字偏大 |
| 100以内加减法 | 100 | 4列 | **16pt** | 三位数可能出现 |
| 乘法/除法 | 100 | 4列 | 自适应 | 根据算式长度计算 |

> 混合选择时，字号取所有选中题型的最保守值。

### 固定参数

| 参数 | 值 |
|------|-----|
| 列数 | 4列 |
| 列宽 | 43.75mm（固定，防止溢出） |
| 列间距 | 5mm |
| 序号格式 | `1、` `2、` …（顿号，竖排序号） |
| 字体 | 宋体（SimSun），标题黑体（SimHei） |
| 页边距 | 上下8mm，左右10mm |
| 纸张 | A4（210×297mm） |
| 对齐 | 表格居中，单元格左对齐 |
| 行高 | 精确计算，均匀填满A4页面 |

### 字号计算逻辑

```
列宽 = (190mm - 3×5mm) / 4 = 43.75mm

估算最宽算式宽度（参考20pt）：
  半角字符 ≈ 4.2mm, 全角字符 ≈ 7mm
  "100、" ≈ 19.6mm
  算式部分 ≈ 字符数 × 4.2mm

字号 = min(22, max(14, 20 × 列宽 / 最宽算式宽度))
```

## 常见问题

**Q: 为什么有些题型题库很小？**
A: 数学限制。如"20以内进位加法"只有约65种不重复组合，这是该规则下的全部可能。系统会自动循环抽取确保多页出题。

**Q: 为什么切换到了原生打印而不是jsPDF？**
A: v2.0曾使用html2canvas+jsPDF直接下载PDF，但存在以下问题：
- 生成的PDF是图片格式，文件体积大
- 手机端html2canvas渲染不稳定，偶发空白
- Blob新窗口+原生打印方案更可靠，且PDF是文本格式（可选文字）

**Q: PDF文件名为什么是"日期+口算题"？**
A: 通过Blob新窗口方案，文档创建时 `<title>` 即为目标文件名，确保所有浏览器（包括Safari和Android Chrome）都能正确命名。

**Q: 题库存在哪里？数据安全吗？**
A: 全部在浏览器本地（内存 + localStorage），不上传任何数据到服务器。
